# 西班牙合同智能Agent — V1.0 开发方案

> 更新时间：2026-05-08
> 目标：核心功能闭环，能完整演示合同生成全流程

---

## 一、V1.0 边界说明

### 包含
- 合同模板上传 + 服务器文件存储 + 元数据入库
- BOE 法律按需抓取 + 文本缓存到 PostgreSQL
- 根据订单信息自动匹配模板 → 获取法律 → 生成完整合同
- 合同预览（Monaco Editor）、编辑、导出（DOCX / PDF）
- 法律问答（WebSocket 流式输出）
- 模型配置（本地 electron-store 管理，支持在线 API / 本地模型）
- 服务器地址配置（存储在 electron-store，随请求发出）
- 前端 Electron 界面覆盖以上所有功能

### 不包含
- 向量数据库 / 嵌入模型（彻底去除）
- 用户注册 / 登录 / JWT 鉴权（V2.0 实现）
- 多租户 / 数据隔离
- 计费 / 用量统计
- 合同审查功能
- 移动端 / Web 端

## 二、客户端本地配置设计（electron-store）

所有客户端配置完全存储在 **Electron 本地**（electron-store 加密），不上传服务器。

```
electron-store 本地加密存储

── 服务器配置 ──
server_url : https://your-server.com   （用户首次填写，指向远端服务器）

── 模型配置 ──
配置A — Anthropic API（默认）
  provider    : anthropic
  api_key     : sk-ant-xxxxxx          （加密存本地，请求时随发，服务器不留存）
  model_id    : claude-opus-4-7
  temperature : 0.1

配置B — 本地 Ollama
  provider    : local
  base_url    : http://localhost:11434/v1
  model_id    : llama3.2
  api_key     : （空）

配置C — DeepSeek（OpenAI 兼容）
  provider    : openai_compatible
  base_url    : https://api.deepseek.com/v1
  api_key     : sk-ds-xxxxxx
  model_id    : deepseek-chat
```

**调用流程：**
前端将 `server_url` + 模型配置随每次请求一起发给后端 → 后端按 `provider` 路由调用对应 AI 服务 → 结果返回前端。**后端数据库中不存任何 API Key 或服务器地址。**

## 六、数据库表（PostgreSQL）

```sql
-- 合同模板元数据
CREATE TABLE contract_templates (
    id              VARCHAR(36) PRIMARY KEY,
    title           VARCHAR(500) NOT NULL,
    category        VARCHAR(100) NOT NULL,   -- EXTRANJERIA / ASESORAMIENTO / GESTORIA / MERCANTIL
    subcategory     VARCHAR(200),
    file_name       VARCHAR(500) NOT NULL,
    file_path       VARCHAR(500) NOT NULL,   -- 服务器路径 /app/storage/templates/...
    raw_text        TEXT NOT NULL,           -- 解析后完整文本（供 Claude 直接读取）
    language        VARCHAR(10) DEFAULT 'es',
    is_active       BOOLEAN DEFAULT TRUE,
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW()
);

-- BOE 法律缓存（按需抓取，避免重复请求 boe.es）
CREATE TABLE cached_laws (
    id          VARCHAR(36) PRIMARY KEY,
    boe_id      VARCHAR(100) UNIQUE NOT NULL,  -- 如 BOE-A-2000-544
    title       VARCHAR(500) NOT NULL,
    category    VARCHAR(100),                  -- 如 extranjeria / civil
    raw_text    TEXT NOT NULL,                 -- 法律完整原文
    source_url  TEXT,
    fetched_at  TIMESTAMP DEFAULT NOW()
);

-- 生成的合同
CREATE TABLE contracts (
    id                VARCHAR(36) PRIMARY KEY,
    title             VARCHAR(500) NOT NULL,
    template_id       VARCHAR(36) REFERENCES contract_templates(id),
    order_input       TEXT NOT NULL,                -- 用户原始订单输入
    extracted_fields  JSONB DEFAULT '{}',           -- Claude 提取的结构化字段
    generated_text    TEXT,                         -- 最终合同全文
    laws_used         JSONB DEFAULT '[]',           -- 引用法律 [{boe_id, title}]
    status            VARCHAR(50) DEFAULT 'draft',  -- draft / confirmed / exported
    export_docx_path  TEXT,                         -- /app/storage/exports/{id}/contract.docx
    export_pdf_path   TEXT,                         -- /app/storage/exports/{id}/contract.pdf
    created_at        TIMESTAMP DEFAULT NOW(),
    updated_at        TIMESTAMP DEFAULT NOW()
);

-- 问答会话
CREATE TABLE chat_sessions (
    id         VARCHAR(36) PRIMARY KEY,
    title      VARCHAR(255),
    created_at TIMESTAMP DEFAULT NOW()
);

-- 消息记录
CREATE TABLE messages (
    id            VARCHAR(36) PRIMARY KEY,
    session_id    VARCHAR(36) REFERENCES chat_sessions(id) ON DELETE CASCADE,
    role          VARCHAR(20) NOT NULL,   -- user / assistant
    content       TEXT NOT NULL,
    input_tokens  INTEGER,
    output_tokens INTEGER,
    created_at    TIMESTAMP DEFAULT NOW()
);
```

## 七、合同生成 LangGraph 工作流（六节点）

```
START
  │
  ▼
节点1: extract_fields
  Claude 解析订单自由文字 → 提取结构化字段
  { nombre, nie, tipo_servicio, honorarios, fecha, ... }
  │
  ▼
节点2: select_template
  从 PostgreSQL 加载所有模板【标题 + 分类 + 子分类】清单
  Claude 选择最匹配的模板 → 返回 template_id
  │
  ├─ 找到 → 继续
  └─ 未找到 → 返回错误，提示用户上传对应模板
  │
  ▼
节点3: load_template
  从 PostgreSQL 读取该模板的完整 raw_text
  │
  ▼
节点4: fetch_laws
  Claude 判断该类合同需要哪些法律 → [Ley 4/2000, RD 557/2011, ...]
  逐一检查 PostgreSQL cached_laws 表：
    有缓存 → 读取 raw_text
    无缓存 → httpx 请求 boe.es → BeautifulSoup4 解析 → 写入 cached_laws
  │
  ▼
节点5: generate_contract
  输入：模板原文 + 法律文本片段 + 结构化字段
  Claude 生成完整合同（PRIMERA / SEGUNDA / TERCERA / CUARTA / QUINTA...）
  │
  ▼
节点6: validate
  检查生成结果是否包含必要条款（PRIMERA、CUARTA 等关键节）
  通过 → END
  不通过 → 重试节点5（最多 2 次，超过则返回错误）
  │
  ▼
END → 保存到 contracts 表（status: draft）→ 返回前端
```

## 八、API 接口

```
系统
  GET    /api/health                       健康检查，返回服务状态

模板管理
  POST   /api/templates/upload             上传 DOCX/PDF，解析文本入库，保存文件到服务器
  GET    /api/templates                    列出所有有效模板
  GET    /api/templates/{id}              模板详情（含 raw_text 预览）
  DELETE /api/templates/{id}             停用模板（软删除，is_active=false）

法律缓存
  GET    /api/laws/boe/search?q=           搜索 BOE，返回候选列表（不入库）
  POST   /api/laws/boe/fetch              抓取并缓存指定法律到 PostgreSQL
  GET    /api/laws                         已缓存法律列表
  DELETE /api/laws/{id}                   删除指定缓存法律

合同生成
  POST   /api/contracts/generate           输入订单文字 → LangGraph 六节点生成合同
  GET    /api/contracts                    历史合同列表
  GET    /api/contracts/{id}              合同详情（含 generated_text）
  PUT    /api/contracts/{id}              编辑合同文本（前端编辑后保存）
  GET    /api/contracts/{id}/export/docx  生成并下载 DOCX（缓存到 export_docx_path）
  GET    /api/contracts/{id}/export/pdf   生成并下载 PDF（缓存到 export_pdf_path）
  DELETE /api/contracts/{id}             删除合同记录及导出文件

问答
  POST   /api/chat/sessions                创建会话
  GET    /api/chat/sessions                会话列表
  WS     /api/chat/{session_id}            流式对话（WebSocket，携带模型配置）
  GET    /api/chat/sessions/{id}/history   历史消息
  DELETE /api/chat/sessions/{id}          删除会话及消息
```

## 十一、环境配置

### backend/.env

```env
# 数据库
DATABASE_URL=postgresql+asyncpg://admin:secret@postgres:5432/contracts_db

# Redis
REDIS_URL=redis://redis:6379

# 文件存储路径（容器内路径）
STORAGE_DIR=/app/storage

# BOE 官网
BOE_WEB_BASE=https://www.boe.es

# CORS（允许 Electron 客户端请求，* 代表全部，生产可收窄）
ALLOWED_ORIGINS=*

# 调试模式
DEBUG=false
```

### docker-compose.yml

```yaml
services:
  postgres:
    image: postgres:16
    restart: unless-stopped
    environment:
      POSTGRES_DB: contracts_db
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: secret
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    restart: unless-stopped

  backend:
    build: ./backend
    restart: unless-stopped
    env_file: ./backend/.env
    depends_on:
      - postgres
      - redis
    volumes:
      - ./backend/storage/templates:/app/storage/templates
      - ./backend/storage/exports:/app/storage/exports
      - ./resources:/app/resources

  nginx:
    image: nginx:alpine
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - backend

volumes:
  postgres_data:
```
